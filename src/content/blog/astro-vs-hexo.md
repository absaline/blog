## 为什么需要静态博客框架？

2026年，建个人博客的选择比以往任何时候都多。但如果你想要：

- **免费托管**（GitHub Pages / Vercel）
- **Markdown 写作**（专注内容，不操心后端）
- **速度快**（静态文件，CDN 分发）
- **零成本**（域名都只需要 $10/年）

那静态博客框架就是最佳选择。

## 为什么我选了 Astro？

三个核心理由：

### 1. Islands 架构

Astro 有个叫 Islands 的概念：页面大部分是纯静态 HTML，只有需要交互的部分才加载 JS。这意味着：
- 首页加载几乎不需要 JS
- 速度碾压 Next.js
- 对 SEO 极其友好

### 2. 零基础友好

你只需要会 Markdown 就能开始写博客。想加交互功能时，可以用 React/Vue/Svelte 组件，不想用也没关系。

### 3. AI 工具配合度最高

- GitHub Copilot 对 `.astro` 文件有完整支持
- Claude 能准确生成 Astro 组件代码
- Vercel AI 能分析构建错误

## 不适合 Astro 的场景

- 需要大量后端逻辑的网站（选 Next.js）
- 追求极致构建速度的超大型站点（选 Hugo）
- 想用纯 Go 技术栈的团队（选 Hugo）

## 小结

对于**零基础个人博客**，2026年我的推荐排序：

1. **Astro** — 最佳综合体验
2. Hugo — 构建速度最快，但学习曲线陡
3. Hexo — 中文生态好，但生态在衰落
